# Project Report Grading Requirements

Source: NUS ISS — Accelerating Digital Excellence

---

## Group Project Report

**Cover:** Project title, Team number and members

### 1. Executive Summary
- Brief overview of project objective and scope
- Key highlights of the solution
- Constraints/assumptions

### 2. System Overview
- Description of overall solution and how agents work together
- High-level workflow diagram

### 3. System Architecture
- Logical architecture diagram with description
- Physical architecture diagram with infrastructure details
- Deployment strategy (e.g., containerization, cloud services)
- Data flow and integration points
- Justification of architectural styles and tech choices

### 4. Agent Roles and Design
For each agent:
- Purpose and responsibilities
- Reasoning patterns
- Planning and memory mechanisms
- Tools used
- Communication protocols and coordination mechanisms

### 5. Explainable and Responsible AI Practices
- How the different stages of development & deployment are aligned with explainable and responsible AI principles
- Approach to fairness, bias mitigation, explainability
- Governance framework alignment (e.g., IMDA's Model AI Governance Framework)

### 6. AI Security Risk Register
- Table of identified risks (e.g., prompt injection)
- Mitigation strategies and security controls

### 7. MLSecOps / LLMSecOps Pipeline
CI/CD diagram with description of:
- Automated testing (including AI security tests)
- Versioning and tracking
- Deployment strategy
- Monitoring and alerting
- Logging and auditability

### 8. Testing Summary
- Types of tests performed (unit, integration, security)
- Results

### 9. Reflection
- Key learnings from the project
- Challenges faced and how they were overcome
- Suggestions for future improvements

---

## Individual Project Report

**Cover:** Project title, Team number, Individual name, Agent responsible for (e.g., "Follow-up Agent")

### 1. Introduction
- Brief overview of the agent's purpose

### 2. Agent Design
- Description of agent role and functionality
- Reasoning and planning workflow
- Memory and state management
- Tools integrated
- Communication and coordination logic
- Prompt engineering patterns used
- Fallback strategies

### 3. Implementation Details
- Summary of implementation approach
- Code structure overview
- Tech stack of the agent (e.g., LLM model, agent framework, MCP etc.)

### 4. Testing and Validation
Types of tests implemented:
- Unit tests
- Integration tests (with other agents or backend services)
- AI security tests (e.g., adversarial inputs, prompt injection handling)
- Results and key findings

### 5. Explainable and Responsible AI Considerations
- How the different stages of development & deployment are aligned with explainable and responsible AI principles
- How this agent addresses explainability
- Bias mitigation strategies (if relevant)
- Handling sensitive content and governance alignment

### 6. Security Practices
- Agent-specific security risks
- Mitigations implemented at code level or workflow level

### 7. Reflection
- Personal learning from implementing this agent
- Suggestions for future improvement of the agent
