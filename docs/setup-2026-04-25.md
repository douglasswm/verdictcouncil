## 2. Environment variables

Two files: `VerdictCouncil_Backend/.env.example` and
`VerdictCouncil_Frontend/.env.example`. Both already exist and are authoritative
— this doc annotates them, it does not replace them. Copy each to `.env` before
running `./dev.sh` (the script auto-copies from the example on first run and
then aborts so you can fill real values).

Legend: **D** = required for local dev, **P** = required for prod/staging,
**C** = required for CI (`.github/workflows/*`).

## 5. Branch protection (configure in GitHub UI)

Apply to **both** submodule repos (`VerdictCouncil_Backend`,
`VerdictCouncil_Frontend`). The orchestration root is trunk-based — no
protection rules needed there.

Protect: `main`, `release/*`, `development`.

Required settings:
- Require pull request before merging; at least **1 approving review**.
- Require branches to be **up to date** before merge.
- Require status checks to pass — check names match `.github/workflows/*` job ids:
  - `lint`, `test`, `security`, `docker` (all present today in `ci.yml`)
  - `eval` — **TBD Sprint 4 D3**, add after the eval gate lands.
- Disallow force-push + branch deletion on protected branches.
- Require linear history on `main` (optional but recommended).

## 6. Quickstart (new contributor)

```bash
# 1. Clone with submodules
git clone --recurse-submodules git@github.com:<org>/VER.git
cd VER

# 2. First boot — dev.sh will copy .env.examples and abort so you can fill values
./dev.sh                      # fails fast, creates .env files
$EDITOR VerdictCouncil_Backend/.env      # set OPENAI_API_KEY, JWT_SECRET
$EDITOR VerdictCouncil_Frontend/.env     # usually fine as-is

# 3. Real boot — brings up Postgres/Redis/MLflow, runs migrations, seeds demo users,
#    starts FastAPI :8001 + arq worker + Vite :5173
./dev.sh

# 4. Smoke-test
open http://localhost:5173    # frontend
curl http://localhost:8001/api/v1/health

# 5. Run backend tests
cd VerdictCouncil_Backend && make test

# 6. Run frontend tests
cd VerdictCouncil_Frontend && npm test

# 7. Stop
# Ctrl+C in the dev.sh terminal stops backend + frontend (infra stays up).
./stop.sh --infra             # stop infra too
```
